Signix — player-tizen (Samsung Tizen Studio)
============================================

Este diretorio e um projeto Web Tizen importavel no Tizen Studio (ficheiros .project, .tproject, .settings e config.xml com perfil tv-samsung).

Importar no Tizen Studio
------------------------
1. Abra o Tizen Studio.
2. File > Import > Tizen > Tizen Project > Next.
3. Escolha "Root directory" e Browse para esta pasta: .../signix-display-pro/player-tizen
4. Next > na lista, marque o projeto "SignixPlayerTizen".
5. Se aparecer aviso de perfil, escolha o perfil Samsung TV (ex.: tv-samsung) e a versao desejada.
6. Finish.

Antes de empacotar
------------------
- Copie js/signix-config.local.example.js para js/signix-config.local.js e preencha supabaseUrl e supabaseAnonKey
  (este ficheiro nao e commitado no git). O index.html faz merge com os valores por defeito.
- Ajuste em config.xml, se necessario: tizen:application id/package (deve coincidir com o certificado de assinatura Samsung) e required_version.

Build e execucao
----------------
- Clique com o botao direito no projeto > Tizen Web Widget / Web Application > Build Signed Package (conforme o menu da sua versao do Studio).
- Run on TV / Emulator: ligue a TV em modo desenvolvedor e use o Device Manager / Run.

Nota: o nome do projeto no Explorador e "SignixPlayerTizen" (definido em .project). Pode renomear no Studio; mantenha package/id alinhados ao certificado.

Pareamento (TV mostra codigo)
-----------------------------
O player Tizen nativo chama as Edge Functions Supabase publicas (anon key):
- create-pairing-code  (POST) — gera linha em public.pairing_codes
- check-pairing-status (POST) — polling ate o painel concluir o claim

Deploy no projeto Supabase (CLI ou Dashboard), por exemplo:
  supabase link --project-ref <REF_DO_PROJECTO>
  supabase functions deploy create-pairing-code
  supabase functions deploy check-pairing-status

Se na TV aparecer «HTTP 404» em create-pairing-code ou check-pairing-status, o URL da TV aponta para um projecto
onde essas funções ainda não foram publicadas (ou o nome não coincide). Resolva com deploy nesse projecto
ou corrija supabaseUrl / anon key em signix-config.local.js para o projecto onde as funções existem.

No painel, o fluxo continua a ser Dispositivos > Novo dispositivo (claimPairingCode), igual ao da pagina /pareamento no browser.

Versao actual do pacote Tizen (widget): ver config.xml (atributo version) e js/constants.js (PLAYER_VERSION_LABEL).

APK Android (nao e Tizen)
-------------------------
Samsung Tizen gera ficheiro .wgt. O APK e o app **Android TV** (Capacitor) na raiz do repo signix-display-pro:
1. Copie .env.capacitor.example para .env.capacitor e defina CAPACITOR_SERVER_URL (HTTPS do site, ex. .../pareamento?platform=android).
2. Na raiz: npm install (se preciso), depois npm run android:live OU npx cap sync android.
3. Abra android/ no Android Studio e Build > Build APK.

Versao Android: android/app/build.gradle (versionCode / versionName) e src/player/version.ts — devem ficar alinhados.
